DNS records, zone transfers, and network host discovery techniques for eJPT reconnaissance.

## DNS Records

| Record | Purpose |
|--------|---------|
| A | Resolves hostname/domain to IPv4 address |
| AAAA | Resolves hostname/domain to IPv6 address |
| NS | Reference to domain's nameserver |
| MX | Resolves domain to mail server |
| CNAME | Domain alias (canonical name) |
| TXT | Text record (SPF, verification, etc.) |
| HINFO | Host information |
| SOA | Start of Authority — domain authority record |
| SRV | Service records |
| PTR | Reverse lookup — IP to hostname |

## DNS Zone Transfer

AXFR (Authoritative Transfer) — replicates entire DNS database from primary to secondary server. Misconfigured servers allow anyone to request full zone data.

```bash
# Using dnsenum (pre-built in Kali)
dnsenum domain.com

# Using dig for zone transfer attempt
dig @ns1.domain.com domain.com AXFR
```

| Tool | Purpose |
|------|---------|
| dnsenum | Automated DNS enumeration + zone transfer |
| dig | Manual DNS queries (AXFR, ANY, etc.) |

## Fierce — DNS Reconnaissance Tool

Semi-lightweight scanner to locate IP space and hostnames against specified domains.

```bash
# Pre-built in Kali
fierce --domain target.com

# With DNS server
fierce --domain target.com --dns-server 8.8.8.8
```

## ICMP / Ping Sweep

Network reconnaissance to identify active hosts by sending ICMP echo requests across an IP range.

Used by admins for network mapping and by attackers to find targets.

---

## Host Discovery with Nmap

```bash
# Ping sweep — discover live hosts (no port scan)
sudo nmap -sn 192.168.1.1/24

# Alternative: netdiscover
sudo netdiscover -r 192.168.1.0/24
```

| Flag | Meaning |
|------|---------|
| -sn | Ping scan only (host discovery, no port scan) |
