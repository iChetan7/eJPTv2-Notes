Passive and active techniques to gather information about a target website before exploitation.

---

## Key Files to Check

| File           | Purpose                                 | What to Look For                                  |
| -------------- | --------------------------------------- | ------------------------------------------------- |
| `/robots.txt`  | Tells crawlers which pages NOT to index | Hidden directories, admin panels, sensitive paths |
| `/sitemap.xml` | Lists all pages for search engines      | Hidden pages, URL structure, attack surface       |

```bash
# Quick checks
curl http://www.domain.com/robots.txt
curl http://www.domain.com/sitemap.xml
```

## WHOIS Enumeration

Passive recon to gather domain ownership, registration details, contact info, name servers.

| Tool | Type | Command/URL |
|------|------|-------------|
| **whois** | CLI (built-in) | `whois domain.com` |
| **NetCraft** | Third-party web | `sitereport.netcraft.com` |

## DNS Reconnaissance

**DNSRecon** — Python-based tool for DNS enumeration (prebuilt in Kali).

```bash
# DNSRecon basic scan
dnsrecon -d domain.com

# Zone transfer attempt
dnsrecon -d domain.com -a

# Full enumeration
dnsrecon -d domain.com -t std,brt
```

| Record Type | Purpose |
|-------------|---------|
| `A` | IPv4 address |
| `AAAA` | IPv6 address |
| `MX` | Mail servers |
| `SOA` | Start of authority |
| `TXT` | Text records (SPF, verification) |
| `SRV` | Service records |

**Online tool:**
```bash
dnsdumpster.com    # Visual DNS recon
```

## WAF Detection — wafw00f

Checks if a website is protected by a Web Application Firewall.

```bash
# Prebuilt in Kali
wafw00f http://www.domain.com

# Multiple targets
wafw00f -i urls.txt
```

## Subdomain Enumeration — Sublist3r

Open-source Python tool for subdomain discovery.

```bash
# Install from GitHub
git clone https://github.com/aboul3la/Sublist3r.git
cd Sublist3r && pip install -r requirements.txt

# Run
python sublist3r.py -d domain.com

# Save output
python sublist3r.py -d domain.com -o subs.txt
```

## Google Dorks (Advanced Search Operators)

| Dork | Purpose |
|------|---------|
| `site:domain.com` | All indexed pages |
| `site:domain.com keyword` | Pages with specific keyword |
| `site:domain.com inurl:admin` | Admin panels |
| `site:domain.com inurl:forum` | Forum pages |
| `site:*.domain.com inurl:admin` | Subdomains with admin |
| `site:*.domain.com intitle:admin` | Admin pages by title |
| `site:*.domain.com filetype:pdf` | PDF files (info disclosure) |
| `intitle:"index of"` | Directory listings |
| `cache:domain.com` | Cached version |
| `inurl:auth_user_file.txt` | Exposed auth files |
| `inurl:passwd.txt` | Exposed password files |

**Database:** `https://www.exploit-db.com/google-hacking-database`

## Email Harvesting — theHarvester

Gathers emails from public sources for **password spray attacks**.

```bash
# Prebuilt in Kali
theHarvester -d domain.com -b all

# Specific sources
theHarvester -d domain.com -b google,linkedin,bing
```

>  **Key Point:** Collected emails = target list for password spray attacks.

## Key Takeaways

| Item | Detail |
|------|--------|
| **Passive recon** | WHOIS, DNS, Google dorks — no direct interaction |
