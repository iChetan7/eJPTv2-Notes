## Nmap Scanning Workflow

Complete scanning methodology from host discovery to UDP enumeration.

---

## 1. Host Discovery

```bash
ping <target-ip>
```

Verify target is alive before scanning.

---

## 2. Default Scan (Top 1000 ports)

```bash
nmap -T4 <target-ip>
```

`-T4` for aggressive timing / faster scan.

---

## 3. Full Port Scan

```bash
nmap -T4 -p- <target-ip>
```

Scans all 65,535 ports. Alternatively, quick-look range for well-known legacy services:

```bash
nmap -T4 -p1-250 <target-ip>
```

---

## 4. Aggressive Scan

```bash
nmap -T4 -A <target-ip>
```

Performs:
- Operating system detection
- Version fingerprinting
- Service info enumeration
- Banner grabbing
- Default NSE scripts
- Traceroute

---

## 5. UDP Scan

```bash
nmap -T4 -sU <target-ip>
```

Scans UDP ports — slower than TCP, often overlooked.

---

## Follow-Up

If any open ports found in previous scans, target them with aggressive scan:

```bash
nmap -T4 -A -p <port1,port2,port3> <target-ip>
```

---

## Flag Reference

| Flag      | Purpose                                               |
| --------- | ----------------------------------------------------- |
| `-T4`     | Aggressive timing template (speed)                    |
| `-p-`     | All 65,535 ports                                      |
| `-p1-250` | Well-known port range                                 |
| `-A`      | Aggressive scan (OS + version + scripts + traceroute) |
| `-sU`     | UDP scan                                              |
| `-p`      | Specific ports to scan                                |
