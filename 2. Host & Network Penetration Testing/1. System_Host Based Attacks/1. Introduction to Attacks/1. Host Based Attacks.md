**Windows Vulnerabilities & Services Overview**

**Why Windows is a Prime Target**

- 70%+ market share → biggest attack surface
- Built in C/C++ → inherits buffer overflow, arbitrary code execution risks
- Not secure by default — needs proactive hardening
- Fragmented versions (XP → 11) still running in the wild
- Patches are slow, many systems left unpatched

**Vulnerability Types (Focus Areas)**

|Type|What it means|
|---|---|
|Information Disclosure|Access to data attacker shouldn't see|
|Buffer Overflow|Overwrite memory → execute malicious code|
|Remote Code Execution|Run code on target remotely|
|Privilege Escalation|Elevate access after initial compromise|
|DoS|Consume resources, crash/freeze system|

---

**Frequently Exploited Windows Services**

|Service|Port|Purpose|
|---|---|---|
|Microsoft IIS|80 / 443|Microsoft's web server|
|WebDAV|80 / 443|File server extension on IIS (upload/delete/move files)|
|SMB|445|File & printer sharing on LAN — **runs by default**|
|RDP|3389|GUI remote access — disabled by default|
|WinRM|5985 / 5986|Remote command execution — disabled by default|

**Key Notes**

- SMB runs by default on all Windows — always check port 445
- RDP and WinRM disabled by default but common in enterprise
- Notable historic exploits: MS08-067 (Conficker), MS17-010 (EternalBlue)
- All services above = native Windows, no third-party software needed
