**Storage**

- Hashes stored in **SAM database** (Security Account Manager)
- SAM is **locked** while OS is running — can't be directly copied
- Attackers dump hashes from **LSASS process memory** instead (e.g. Mimikatz)
- Modern Windows encrypts SAM with a **syskey**
- Need **admin/elevated privileges** to access LSASS

![[Pasted image 20260504194628.png]]

**Key Takeaways**

- You'll almost **never see LM** hashes in real engagements
- **NTLM** is what you'll encounter on all modern Windows
- No salts = vulnerable to **brute force & rainbow table** attacks
- Tools covered next: **Mimikatz**, Windows Credential Editor
