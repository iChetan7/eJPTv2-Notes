**How Linux Stores Passwords:**

- `/etc/passwd` — stores user account info (readable by all users, no passwords)
- `/etc/shadow` — stores actual encrypted password hashes (root only)
- Must have **root/elevated privileges** to access shadow file

**Hash Format in /etc/shadow:**

```
username:$<id>$<salt>$<hash>
```

**Hash Algorithm Identifier:**

|Value|Algorithm|Strength|
|---|---|---|
|`$1`|MD5|Weak — easy to crack|
|`$2`|Blowfish|Weak — easy to crack|
|`$5`|SHA-256|Strong|
|`$6`|SHA-512|Strongest — modern systems|

**Linux vs Windows hashes:**

- Windows NTLM hashes → can be used for Pass-the-Hash attack
- Linux hashes → can only be cracked, not reused for auth

**⚠️ Key Points**

- Shadow file needs **root access** — elevate privileges first
- `$6$` = SHA-512 — most common on modern Linux
- Unshadowed file = passwd + shadow combined — format ready for cracking
- Linux hashes next step = password cracking (Hashcat/John)
- `setg RHOSTS` = sets global variable — no need to set RHOSTS every time 🔥
