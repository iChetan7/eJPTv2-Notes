**What & Why**

- TShark = CLI version of Wireshark
- Better for targeted analysis, automation, scripting
- GUI Wireshark = manual browsing; TShark = fast, automated extraction

---

**Basic Commands**

```bash
# version check
tshark -v

# help menu
tshark -h | more

# list interfaces
tshark -D

# capture on interface (needs root)
tshark -i eth0

# read a pcap file
tshark -r file.pcap

# count packets
tshark -r file.pcap | wc -l
```

**Statistics**

```bash
# protocol hierarchy (like Wireshark's Statistics menu)
tshark -r file.pcap -z io,phs -q

# conversations
tshark -r file.pcap -z conv,tcp -q

# HTTP requests
tshark -r file.pcap -z http,tree -q
```

**Filtering**

```bash
# filter by protocol
tshark -r file.pcap -Y "http"
tshark -r file.pcap -Y "tcp"
tshark -r file.pcap -Y "dns"

# filter by IP
tshark -r file.pcap -Y "ip.addr == 192.168.1.1"

# filter by port
tshark -r file.pcap -Y "tcp.port == 80"
```

**Key Notes**

- Wireshark = visual browsing of packets
- TShark = when you know what you're looking for
- Can be scripted into automated tools
