*What is TShark?**

- Command line version of Wireshark
- Same filters, no GUI
- Better for remote/scripted use — quick and lightweight
- Read existing PCAP files or capture live traffic

---

## Commands

**Basic TShark syntax**

```bash
tshark -r <file.pcap> -Y 'filter'
```

**View only HTTP traffic**

```bash
tshark -r file.pcap -Y 'http' | more
```

**Filter by source and destination IP**

```bash
tshark -r file.pcap -Y 'ip.src == 192.168.252.128 && ip.dst == 52.32.74.91'
```

**Filter only HTTP GET requests**

```bash
tshark -r file.pcap -Y 'http.request.method == GET' | more
```

**Show specific fields only (time, source IP, URI)**

```bash
tshark -r file.pcap -Y 'http.request.method == GET' -T fields -e frame.time -e ip.src -e http.request.uri
```

**Find packets containing keyword**
 
```bash
tshark -r file.pcap -Y 'http contains password'
```

**Find traffic to specific site**

```bash
tshark -r file.pcap -Y 'http.request.method == GET && http.host contains "nytimes.com"' -T fields -e ip.dst
```

**Extract HTTP cookies (session IDs)**

```bash
tshark -r file.pcap -Y 'ip contains "amazon.in"' -T fields -e http.cookie
```

**Find user agent string (browser/OS info)**

```bash
tshark -r file.pcap -Y 'ip.src == 192.168.252.128 && http' -T fields -e http.user_agent
# Returns: Mozilla/5.0 (X11; Linux x86_64)...
```

---

**Filter Syntax Reference**

| Operator    | Meaning                     |
| ----------- | --------------------------- |
| `==`        | equals                      |
| `&&`        | AND                         |
| `\|`        | OR                          |
| `contains`  | string match inside packet  |
| `-T fields` | output specific fields only |
| `-e`        | specify which field to show |

---

**⚠️ Key Points**

- Always use matching quotes — single or double, never mix
- `contains` searches anywhere inside the packet data
- User-Agent string reveals OS + browser of target
- Session/cookie IDs from HTTP traffic can be used for session hijacking
- HTTP = readable, HTTPS = encrypted/unreadable
- Use `| more` to paginate large outputs 🔥
