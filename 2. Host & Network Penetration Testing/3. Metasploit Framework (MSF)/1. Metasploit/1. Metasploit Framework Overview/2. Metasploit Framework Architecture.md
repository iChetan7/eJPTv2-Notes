## Module Types

| Module        | Purpose                                                               |
| ------------- | --------------------------------------------------------------------- |
| **Exploit**   | Takes advantage of a vulnerability; always paired with a payload      |
| **Payload**   | Code executed on the target after exploitation (gives shell/access)   |
| **Encoder**   | Obfuscates payloads to evade antivirus detection                      |
| **NOPs**      | Ensures payload size consistency and stability; prevents crashes      |
| **Auxiliary** | Scanning, enumeration, fuzzing — cannot be paired with a payload      |
| **Post**      | Post-exploitation tasks (privilege escalation, hash extraction, etc.) |

## Payload Types

**Non-staged** — entire payload sent to the target in one go alongside the exploit.

**Staged** — split into two parts:

- **Stager** (part 1): establishes the reverse connection back to attacker
- **Stage** (part 2): downloaded via that connection; delivers the actual shell or Meterpreter session

## Meterpreter

The most powerful MSF payload. Key characteristics: runs entirely **in memory** (no disk traces), communicates over a stager socket, and provides a full interactive session with capabilities including file system navigation, file upload/download, keylogging, privilege escalation, and hash extraction.

## File System Structure (Kali Linux)

```
/usr/share/metasploit-framework/
└── modules/
    ├── auxiliary/
    │   └── scanner/ (ftp, smb, discovery, etc.)
    ├── exploits/
    │   └── (android, linux, windows, multi, etc.)
    ├── payloads/
    ├── encoders/
    │   └── x86/shikata_ga_nai
    ├── nops/
    └── post/

~/.msf4/modules/   ← custom/user modules go here
```

Exploit modules are organized by **target OS** (Linux, Windows, Android, iOS, etc.) and then by **service** (SMB, FTP, MySQL, browser, etc.), making it easy to locate the right module quickly.
