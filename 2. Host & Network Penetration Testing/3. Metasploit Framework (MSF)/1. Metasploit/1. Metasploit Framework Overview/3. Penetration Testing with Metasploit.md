## The PTES Framework

The **Penetration Testing Execution Standard (PTES)** is the methodology used as a roadmap, developed by infosec practitioners to standardize how penetration tests are conducted.

## Pentest Phases & Metasploit Integration

| Phase                      | Objective                                | Metasploit Role                                       |
| -------------------------- | ---------------------------------------- | ----------------------------------------------------- |
| **Information Gathering**  | Enumerate OS, services, versions, config | Auxiliary modules + Nmap (import results into MSF DB) |
| **Vulnerability Scanning** | Identify exploitable weaknesses          | Auxiliary modules + Nessus (import results into MSF)  |
| **Exploitation**           | Gain initial access                      | Exploit modules + payloads (Windows & Linux targets)  |
| **Post-Exploitation**      | Operate on compromised system            | Meterpreter session                                   |
| **Privilege Escalation**   | Elevate to highest privilege level       | Meterpreter + post-exploitation modules               |
| **Persistent Access**      | Maintain access if session drops         | Meterpreter + persistence modules                     |
| **Clearing Tracks**        | Remove evidence of compromise            | Clear Windows Event logs, bash history, etc.          |

## Key Takeaway

Metasploit isn't just an exploitation tool —> it integrates across the **entire pentest lifecycle**, from initial reconnaissance through post-exploitation. External tools like Nmap and Nessus feed results into MSF's database, making it a central hub for the engagement.
