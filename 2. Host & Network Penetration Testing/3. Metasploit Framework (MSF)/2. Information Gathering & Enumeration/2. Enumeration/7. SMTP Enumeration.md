## What is SMTP?

Simple Mail Transfer Protocol — used for email transmission. Default port is **TCP 25**, but can run on port **465 or 587** when SSL/TLS is configured.

## SMTP Enumeration Workflow

**Step 1 — Identify SMTP version**

```bash
use auxiliary/scanner/smtp/smtp_version
# RPORT defaults to 25
run
# Result: Postfix SMTP + domain/email address of the server
```

**Step 2 — Enumerate user accounts**

```bash
use auxiliary/scanner/smtp/smtp_enum
set USER_FILE /usr/share/metasploit-framework/data/wordlists/unix_users.txt
run
# Result: admin, administrator, backup, bin, daemon, www-data, etc.
```

## How SMTP User Enumeration Works

SMTP has two built-in commands that reveal user information:

| Command | Purpose                                                   |     |
| ------- | --------------------------------------------------------- | --- |
| `VRFY`  | Confirms whether a username is valid on the server        |     |
| `EXPN`  | Reveals actual addresses behind aliases and mailing lists |     |

These commands, when enabled, allow attackers to verify which usernames exist on the system.

## Why SMTP Enumeration Matters

SMTP itself is rarely exploited directly, but the **usernames enumerated from it** are valuable for staging other attacks. For example, if SSH is also running on the same target, those usernames can be fed directly into `ssh_login` for a targeted brute force much faster than guessing both username and password blindly.
