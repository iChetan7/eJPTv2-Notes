## What is Armitage?

A **free Java-based GUI frontend** for the Metasploit Framework, developed by **Raphael Mudge**.

| Feature            | Description                                                    |
| ------------------ | -------------------------------------------------------------- |
| **Purpose**        | Simplifies network discovery, exploitation & post-exploitation |
| **Alternative to** | `msfconsole` (provides same functionality in GUI)              |
| **Key Advantage**  | **Visualizes targets, exploits, and pivoting** in real-time    |
| **Automation**     | Single-click port scanning, exploitation, post-exploitation    |
| **Availability**   | Pre-packaged with Kali Linux & pentest distributions           |

## Prerequisites & Setup

### Required Services
```bash
# 1. Start PostgreSQL database
service postgresql start

# 2. Start msfconsole (starts backend services)
msfconsole

# 3. Verify database connection
db_status
# Output: connected to MSF, connection type: PostgreSQL
```

### Launching Armitage
```bash
# Method 1: Terminal
armitage

# Method 2: GUI Menu
# Applications > Exploitation Tools > Armitage
```

**Connection Dialog:**
- Use default Metasploit database credentials
- Click **Connect**
- If prompted "Start Metasploit RPC server?" → **Yes**

## Interface Layout

### Three Main Panes

| Pane | Location | Purpose |
|------|----------|---------|
| **Module Menu** | Left | Browse/search modules (Auxiliary, Exploits, Payloads, Post) |
| **Target View** | Center | Visual representation of hosts (icons change by OS) |
| **Console** | Bottom | Embedded `msfconsole` — runs jobs in tabs |

### Toolbar Menus

| Menu | Key Functions |
|------|---------------|
| **Armitage** | Preferences, target view (graph/table), SOCKS proxy, listeners |
| **View** | Credentials, downloads, jobs, loot, activity logs |
| **Hosts** | Import/add hosts, **Nmap scan**, DNS enumeration |
| **Attacks** | Vulnerability scanning, **Find Attacks**, **Hail Mary** ⚠️ |
| **Workspaces** | Manage session tabs |

> ⚠️ **Avoid "Hail Mary"** — throws all applicable exploits at target indiscriminately

## Port Scanning Workflow

### Step 1 — Add Target Host
```
Hosts > Add Host
→ Paste target IP address
→ Click Add
→ Host appears as computer icon in target pane
```

### Step 2 — Basic TCP Scan (Quick)
```
Right-click host > Scan
```
- Uses: `auxiliary/scanner/portscan/tcp`
- Auto-sets: `RHOSTS`, threads=24, ports=1-50000
- Opens progress tab in console

### Step 3 — Nmap Scan (Recommended for Enumeration)
```
Hosts > Nmap Scan > Quick scan plus OS detection
```

### Available Nmap Scan Types:

- **Intense scan** → Deep service detection
- **Intense scan + UDP** → TCP + UDP ports  
- **Intense scan, all TCP ports** → Full TCP range (slow)
- **Ping scan** → Host discovery only
- **Quick scan** → Fast top ports
- **Quick scan + OS detection** → Best balance — RECOMMENDED

**After Nmap completes:**
- Message: *"Use attacks to find applicable exploits"*
- Host icon updates based on detected OS (Windows icons vary by version)
- Services populated with version info

### Step 4 — View Enumerated Services
```
Right-click host > Services
```
Shows: Port | Protocol | State | Service | Version Info

> 💡 Equivalent to running `services` in `msfconsole`

## Host Management

### Labeling Hosts
```
Right-click host > Host > Set Label
→ Example: "Victim 1"
```

### Manual OS Setting
```
Right-click host > Host > Set Operating System
```

## Manual Module Execution

```
1. Search in Module pane (e.g., "port scan")
2. Click module name
3. Configure options (double-click values to edit):
   - RHOSTS, LHOST, LPORT
   - Target-specific settings
   - Payload selection
4. Click Launch
```

## Key Takeaways

| Tip | Details |
|-----|---------|
| **Performance** | Java-based — can be slow; run **one function at a time** |
| **Workspace hygiene** | Close tabs when done to avoid clutter |
| **Visualization** | Host icons change based on OS detection accuracy |
| **Database integration** | All scans sync with Metasploit PostgreSQL DB automatically |
| **Find Attacks** | After Nmap, use `Attacks > Find Attacks` to suggest exploits |
| **No Attack menu?** | If no exploits found, perform manual vulnerability identification |
