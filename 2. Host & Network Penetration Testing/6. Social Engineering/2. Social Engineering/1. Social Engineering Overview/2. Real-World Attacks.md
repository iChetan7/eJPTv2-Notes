A walkthrough of **actual social engineering attacks** that occurred in the wild, analyzing the tactics used, the impact, and the lessons learned for penetration testing.

---

## Case Study 1: Google & Facebook Fake Invoicing Scam (2013–2015)

| Attribute | Detail |
|---|---|
| **Target** | Google and Facebook |
| **Method** | Impersonated electronics manufacturer, sent fake invoices with forged executive signatures |
| **Preparation** | Reconnaissance: identified supplier relationships, invoice formats, obtained signatures |
| **Loss** | Hundreds of millions of USD |
| **Outcome** | Culprit arrested, funds recovered |

**Key Takeaway**: Deep reconnaissance + forged documents = massive financial fraud. Social engineering isn't just emails — it's the **entire lifecycle of information gathering**.

## Case Study 2: FACC CEO Fraud (2016)

| Attribute | Detail |
|---|---|
| **Target** | FACC (Aircraft manufacturing company, China) |
| **Method** | Fake president fraud — compromised/impersonated CEO email |
| **Tactic** | Urgent wire transfer request sent when CEO was unavailable to verify |
| **Loss** | €54 million |
| **Outcome** | CEO and CFO fired; company sued them; partial recovery via insurance |

**Attack Flow:**
```text
Attacker trolls company → Information gathering
    ↓
Sends urgent phish: "Wire transfer now / Pay time-sensitive invoice"
    ↓
Victim feels: Urgency + Fear of screwing up deal + Toxic work culture
    ↓
Fraudulent wire transfer executed
```

**Key Takeaway**: BEC relies on **timing, authority, and workplace pressure**. The human cost extends beyond money — careers are destroyed.

## Case Study 3: Robinhood Vishing Scam (Nov 2021)

| Attribute | Detail |
|---|---|
| **Method** | Vishing (voice phishing) via customer service line |
| **Duration** | ~2 hours on the phone |
| **Data Stolen** | 5M+ email addresses, 200 full names, 300 birthdays + zip codes |
| **Discovery** | Customer service rep told a friend → friend flagged it |

**Why it worked**: Long phone call built trust; attacker escalated case gradually; support staff not trained to verify identity rigorously.

**Key Takeaway**: **Vishing is data aggregation** — even "harmless" pieces (email + name + DOB + zip) enable mass identity fraud.

## Case Study 4: Fake Excel File — HTML Malware (April 2021)

| Attribute | Detail |
|---|---|
| **Disguise** | `.xls` / `.xlsx` file that is actually HTML |
| **Delivery** | "Here is the invoice" / "Here is the bookkeeping file" |
| **Execution** | User opens with browser → reaches attacker's server |
| **Payload** | Fake Office 365 login page → credentials harvested |

**Technical Trick:**
```text
File: invoice.xls (actually HTML inside)
    ↓
Opens in browser (not Excel)
    ↓
Downloads payload from attacker server
    ↓
Renders fake Office 365 login
    ↓
User enters credentials → sent to attacker
    ↓
BEC / escalation / full compromise
```

**Key Takeaway**: **File extension spoofing** bypasses user judgment. Always verify file type, not just extension.

## Case Study 5: HTML Table Windows Logo — Filter Bypass

| Attribute | Detail |
|---|---|
| **Problem** | Email filters flag emails containing image files (logos) |
| **Bypass** | Recreate Windows logo using **HTML table cells** (4-cell colored table) |
| **Result** | No image file = no filter flag = email delivered |

**Visual:**
```
┌─────────┬─────────┐
│  BLUE   │  GREEN  │  ← HTML table with colored cells
├─────────┼─────────┤
│  RED    │ YELLOW  │  ← Looks like Windows logo to human eye
└─────────┴─────────┘
```

**Key Takeaway**: Filters look for **images** — attackers use **HTML/CSS** to replicate visuals. Defense must go beyond simple attachment scanning.

## Case Study 6: FIN7 USB Baiting (Aug–Nov 2021)

| Attribute | Detail |
|---|---|
| **Delivery** | Physical packages via UPS/USPS |
| **Impersonation** | HHS (Health & Human Services), Amazon |
| **Contents** | USB stick with malware |
| **Social Hook** | "COVID info from HHS" or "Gift card — plug in USB to redeem" |
| **Malware** | Auto-run malware, likely reverse shell + keylogger |

**Attack Chain:**
```text
Package arrives (trusted brand)
    ↓
User plugs in USB (curiosity/expectation)
    ↓
Auto-run executes malware
    ↓
Reverse shell established / keylogger installed
    ↓
Lateral movement, data exfiltration
```

**Key Takeaway**: **Physical media + trusted brand impersonation** bypasses all network-level defenses. USB auto-run policies are critical.

## Why These Matter for Pen Testing

| Pen Test Objective | How These Cases Inform It |
|---|---|
| **Test user awareness** | Can employees spot fake invoices, vishing, or USB drops? |
| **Test email filters** | Do filters catch HTML tables, spoofed extensions, or logo-based bypasses? |
| **Test incident response** | How quickly is social engineering detected and reported? |
| **Test physical security** | Will staff challenge strangers or plug in found USBs? |
