How to **formally integrate social engineering** into a structured penetration test — covering scoping, information gathering, external access, and physical access techniques, all within the bounds of the Rules of Engagement (RoE).

---

## Why SE in Pen Testing?

| Standard | Relevance |
|---|---|
| **NIST SP 800-115** | Includes social engineering as a formal pen test component — vishing, BEC, and phishing to test organizational controls |
| **Regulatory Compliance** | Many frameworks require proof that human-layer controls are tested, not just technical ones |

> The goal: **Find if controls work** — not to break everything or cause harm.

---

## Critical: Scope Matters

Social engineering edges into what people find **ethically uncomfortable** even when legal. Always stay within the contract.

| Scope Example | Interpretation |
|---|---|
| *"Anything short of kidnapping the CEO"* | Very broad — requires extreme care and legal review |
| *"Phishing only, no physical access"* | Narrow — stick to email-based SE |
| *"No vishing after hours"* | Time-bound restriction |

**Golden Rule**: If it's not in scope, don't do it — even if it would "work."

## SE for Information Gathering (OSINT + Human Intel)

### DEFCON Social Engineering Village
Competitors gather intel via phone calls using pretexting. Winner (2019, Darknet Diaries) impersonated IT support:

```text
Pretext: "We're sending you a new computer. 
         Need to know your current model for the checklist."
Result: Detailed system info, network details, employee names
```

### Passive Physical Recon
| Location | What to Observe |
|---|---|
| **Coffee shops near office** | Employee chatter, badge types, dress code, lingo |
| **Smoke pits** | Badge designs, break schedules, meeting times, network usage patterns |
| **Parking lots** | Employee arrival/departure times, vehicle stickers, building access points |

> **Key Insight**: People with their guard down reveal more than they realize.

## SE for External Access

| Vector | Method | Goal |
|---|---|---|
| **Phishing** | Malicious email with reverse shell/downloader | Initial foothold |
| **Credential Harvesting** | Fake O365/G Suite login page | Cloud access (SaaS compromise) |
| **Malicious Attachments** | Fake Excel (HTML inside), PDF exploit | Code execution |
| **Watering Hole** | Compromise website frequented by employees | Drive-by download |

**Target**: Bypass the DMZ and firewall by compromising a human with internal access.

## SE for Physical Access

### Badge Cloning
```text
Tool: Proxmark3 (RFID reader/cloner)
Pretext: "The reader is malfunctioning. Need to test it — can I scan your badge?"
Location: Smoke pit (hurried employees, low guard)
Result: Clone badge → same access as employee
```

### USB Baiting (Rubber Duckies)
| Variant | Description |
|---|---|
| **Generic USB** | Plain drive left in parking lot |
| **Branded USB** | Company logo printed on it |
| **Curiosity hook** | "Confidential" or "Payroll" written on it |
| **Proxmox/ETH attack** | USB-to-Ethernet adapter — inline between PC and wall jack; auto-establishes reverse shell to C2 |

### Physical Impersonation
| Prop/Tactic | Why It Works |
|---|---|
| **Ladder** | "Obviously going somewhere to fix something" — no one challenges |
| **Blue folder / clipboard** | Looks official; implies belonging |
| **Service worker uniform** | Cable guy, plumber, electrician — with inside info ("Mr. Miller, 3rd floor, toilet repair") |
| **Tailgating at rush hour** | "Forgot my badge" — everyone sympathetic, security overwhelmed |

### Unlocked Workstation Attack
```text
Scenario: Employee goes to bathroom, leaves PC unlocked
Action: Plug in rubber ducky (3 seconds) → reverse shell established
        OR plug in USB-ETH adapter inline → persistent C2 access
Result: Full network access without ever "hacking" a system
```

## Physical SE Controls to Test

| Control | What to Verify |
|---|---|
| **USB policies** | Auto-run disabled? USB ports physically blocked? |
| **Badge checks** | Are strangers challenged? Tailgating prevented? |
| **Visitor procedures** | Guest badges? Escort requirements? |
| **Workstation lock** | Screen lock timeout? Policy enforcement? |
| **Security awareness** | Do employees report suspicious people/devices? |

## Summary: SE in Pen Testing

| Phase | SE Technique | Purpose |
|---|---|---|
| **Reconnaissance** | Coffee shop/smoke pit observation, vishing | Gather intel for targeted attacks |
| **External Access** | Phishing, credential harvesting, watering hole | Bypass perimeter defenses |
| **Physical Access** | Badge cloning, USB drops, impersonation, tailgating | Gain internal network access directly |

---

## Key Takeaways

| Item | Detail |
|---|---|
| **Legal boundary** | Scope = contract = law. Never exceed it. |
| **Pretexting is everything** | The backstory makes or breaks the attack |
| **Trust is the vulnerability** | People want to help — exploit that psychology |
| **Physical access trumps all technical controls** | One unlocked workstation = full compromise |
| **Test the controls, don't just break in** | Report gaps so the organization can fix them |
