A hands-on walkthrough of setting up and executing a **controlled phishing campaign** using GoPhish — an open-source phishing framework written in Go. This demonstrates how penetration testers simulate real-world phishing attacks to test organizational resilience.

---

## Lab Environment

| Component | Detail |
|---|---|
| **Tool** | GoPhish (open-source phishing framework) |
| **Admin Interface** | `https://127.0.0.1:3333` |
| **SMTP Server** | `localhost:25` |
| **Landing Page** | `localhost:8080` (cloned INE password reset page) |
| **Victim Email** | `victim@demo.ine.local` |
| **Note** | No SSL certificates — accept the security risk in lab |

---

## Step 1 — Start GoPhish Server

```bash
# Launch GoPhish (pre-configured in lab)
./gophish

# Output:
# Admin server: https://127.0.0.1:3333
# Phish server: http://0.0.0.0:80
```

> Access admin panel in Firefox → `https://127.0.0.1:3333` → Accept risk → Login with credentials from instructions.

---

## Step 2 — Configure Sending Profile

```bash
# In GoPhish UI: Sending Profiles → New Profile

Name: red
From: support@demo.ine.local      # Spoofed sender
Host: localhost:25                 # Local SMTP server
Username: red@demo.ine.local
Password: <from_instructions>
Ignore Certificate Errors: ✓

# Test it:
Send Test Email → victim@demo.ine.local
# Verify: Victim's inbox receives email from "support@demo.ine.local"
```

---

## Step 3 — Set Up Landing Page

```bash
# Target: Clone INE password reset page
# Lab has local copy at: localhost:8080

# In GoPhish: Landing Pages → New Page
Name: INE Password Reset
URL: http://localhost:8080        # Local clone for lab
Capture Submitted Data: ✓         # Enable credential harvesting
Capture Passwords: ✓             # Capture passwords too
Redirect to: http://localhost:8080  # After submission, loop back
```

> **Real-world**: Use a cloned page hosted on your own domain/Cloud server. Ensure it looks identical to the legitimate site.

---

## Step 4 — Create Email Template

```bash
# Source: Real .eml file saved from inbox → renamed to .txt
# Contains HTML tables, headers, branding — looks authentic

# In GoPhish: Email Templates → New Template
Name: INE Password Reset
Subject: Password Reset Required

# Import Email → Paste entire .eml content
# Replace personal details with variables:
#   {{.FirstName}} → auto-populates from target list
#   {{.URL}} → auto-inserts phishing link

# Enable: Add Tracking Image ✓
# This embeds a 1x1 pixel image that calls back to GoPhish server
# when email is opened — reveals "Email Opened" status
```

---

## Step 5 — Import Targets (Users & Groups)

```bash
# CSV format: First Name, Last Name, Email, Position
# Pre-loaded targets:
#   Alice Inchains (CEO)
#   Bob Barker (CFO)
#   John Wayne (CMO)
#   Vic Tim (Intern)

# In GoPhish: Users & Groups → New Group
Name: targets
Import CSV: targets.csv
```

---

## Step 6 — Launch Campaign

```bash
# In GoPhish: Campaigns → New Campaign

Name: INE password reset for target company
Email Template: INE Password Reset
Landing Page: INE Password Reset
URL: http://localhost            # Where landing page is hosted
Sending Profile: red
Groups: targets

# Launch Campaign → Emails sent with slight random delay (~5ms apart)
```

## Key GoPhish Features

| Feature | Purpose |
|---|---|
| **Tracking Image** | 1x1 pixel beacon — detects email opens without user interaction |
| **Variable Substitution** | `{{.FirstName}}`, `{{.URL}}` — personalizes emails at scale |
| **Credential Capture** | Landing pages can harvest usernames/passwords |
| **Campaign Scheduling** | Stagger emails over time to avoid detection |
| **Export Reports** | CSV export for client reporting |

---

## Real-World Considerations

| Aspect | Guidance |
|---|---|
| **Domain Setup** | Register lookalike domains (typosquats) — takes days to configure properly |
| **Scope** | Must be explicitly authorized in pen test contract |
| **Legal Review** | Consult lawyers — some phishing simulations have backfired (e.g., fake bonus emails) |
| **Landing Page Realism** | Clone legitimate sites exactly — but ensure it's distinguishable to avoid real harm |
| **Team Collaboration** | GoPhish supports multi-user; webhooks enable real-time integration |

---

## Key Takeaways

| Item | Detail |
|---|---|
| **GoPhish is reusable** | Set up once, tweak per campaign |
| **Backend setup takes days** | Domain, SMTP, hosting, SSL — not a 5-minute job |
| **Tracking works silently** | 1-pixel image reveals opens without victim knowledge |
| **Credential harvesting** | Enable only if explicitly in scope |
| **Report everything** | Export CSV for client deliverables |
