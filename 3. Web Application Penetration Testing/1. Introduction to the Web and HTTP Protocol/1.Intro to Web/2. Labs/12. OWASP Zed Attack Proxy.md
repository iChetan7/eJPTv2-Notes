## What is ZAP (OWASP Zed Attack Proxy)?
A free, open-source web application security scanner and proxy. **ZAP** is a beginner-friendly alternative to Burp Suite, featuring a built-in **HUD (Heads Up Display)** for in-browser testing. It's excellent for manual exploration, fuzzing, and automated scanning of web apps like bWAPP.

---

## Attack Flow: HTTP Login Fuzzing with ZAP

### Step 1 — Setup & Manual Explore
```bash
# Launch ZAP and start Manual Explore
# Target: bWAPP running on http://<target-ip>/
```

In ZAP GUI:
- Click **Manual Explore**
- Enter URL: `http://<target-ip>/`
- Enable **Use the HUD** → Launch Browser

---

### Step 2 — Capture the Login Request
1. Navigate to the bWAPP login page via HUD
2. Enter dummy credentials: `test` / `test`
3. Submit login (will show "Invalid credentials")
4. In ZAP, find the **POST /login.php** request in the History panel
5. Right-click the request → **Open/Resend with Request Editor**

---

### Step 3 — Configure Fuzzing
Right-click the captured POST request → **Fuzz...**

**Payload 1 — Username field:**
- Highlight `test` (username value)
- Click **Add** → **Add...**
- Enter payloads manually:
  - `admin`
  - `bee`

**Payload 2 — Password field:**
- Highlight `test` (password value)
- Click **Add** → **Add...**
- Enter payloads manually:
  - `admin`
  - `password`
  - `adminpassword`
  - `cookie`
  - `hello`
  - `cookie1`
  - `bug`
  - `bee`

> **Note:** You can also load a full wordlist file instead of manual entry.

---

### Step 4 — Run the Fuzzer
```bash
# In ZAP Fuzzer dialog:
# - Verify both payload positions are set
# - Click Start Fuzzer
```

|Status Code|Meaning|Indicator|
|:--|:--|:--|
|`200 OK`|Login failed, page returned normally|Invalid credentials|
|`302 Found`|Redirect after successful login|**Valid credentials found!**|

---

### Step 5 — Identify Valid Credentials
From fuzzer results:
- Look for the **302 redirect** among mostly `200` responses
- In this lab: **username = `bee`, password = `bug`** triggered the 302
- Verify manually: log in with `bee` / `bug` → success, logged in as `bee`

## Key Points

| Item | Detail |
|------|--------|
| **Tool** | OWASP ZAP (with HUD enabled) |
| **Target** | bWAPP login form (HTTP POST) |
| **Technique** | Form-based fuzzing / credential brute-force |
| **Success indicator** | `302 Found` redirect (vs `200 OK` on failure) |
| **Valid credentials found** | `bee` / `bug` |
| **Alternative to** | Burp Suite (ZAP is free and more beginner-friendly) |
| **HUD feature** | In-browser overlay for quick testing without switching windows |

---

## Key Takeaways
- **ZAP's Fuzzer** lets you brute-force login forms by replacing multiple payload positions simultaneously
- **HTTP 302** after a login POST almost always means successful authentication
- **Manual payload entry** works for small tests; use wordlists for real engagements
- **HUD mode** is great for beginners — less overwhelming than Burp Suite's full interface
- Always check the **Fuzzer tab** for status code differences to spot valid credentials quickly
