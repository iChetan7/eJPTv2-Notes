## What is this lab about?
A hands-on lab demonstrating **directory enumeration using Burp Suite Community Edition's Intruder module**. This covers setting up the proxy, configuring Sniper attack mode, loading wordlists, and interpreting results — with emphasis on Burp's throttling limitations in the free version.

---

## Key Concepts Explained

**Burp Suite Intruder Attack Modes**
- **Sniper** — Single payload position, one value at a time (best for single variable fuzzing).
- **Battering Ram** — Same payload across all positions simultaneously.
- **Pitchfork** — Multiple payload sets, one per position, parallel iteration.
- **Cluster Bomb** — Multiple payload sets, all combinations tested (cartesian product).

**Why Burp Suite for Directory Busting?**
- Industry standard for web app pentesting — professionals use it for bug bounties and engagements.
- All-in-one environment: proxy, repeater, intruder, spider — no tool switching needed.
- **Community Edition limitation**: Throttled speed (much slower than Gobuster/DIRB).

## Key Takeaways

| Finding | Status | Notes |
|---|---|---|
| `/data` | 301 | Redirects to `/data/` — directory exists |
| `/passwords` | 301 | Redirects to `/passwords/` — directory exists |
| `/uploads` | 301 | Confirmed from previous labs |
| Throttling | N/A | CE limits speed; Pro version removes this |

- **Sniper mode** is ideal for single-position fuzzing like directory names.
- **301 status** on directories without trailing slash = Apache/nginx redirect behavior.
- **Burp CE throttling** makes it impractical for large wordlists — use Gobuster/DIRB for speed, Burp for precision.
- **Professional workflow**: Start in Burp, use it for everything (proxy → spider → intruder → repeater) rather than switching tools.
- **Payload markers (§)** define where Intruder injects values — clear defaults first, then add your own.
