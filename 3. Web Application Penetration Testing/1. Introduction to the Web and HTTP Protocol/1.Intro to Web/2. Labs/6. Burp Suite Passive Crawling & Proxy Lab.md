## What is this lab about?
A hands-on lab demonstrating **Burp Suite's proxy and passive crawling capabilities**. Covers setting up the proxy with FoxyProxy, intercepting requests, passive site mapping through browsing, HTTP history analysis, and scope management to stay within engagement boundaries.

---

## Key Concepts Explained

**Burp Suite Proxy**
- Intercepts all HTTP/HTTPS traffic between browser and target.
- Allows inspection, modification, and replay of requests.
- **Passive crawling** — Burp automatically maps the application as you browse without sending additional requests.

**Scope Management**
- Defines what targets Burp will actively process and attack.
- Out-of-scope traffic is ignored — critical for staying within legal boundaries.
- Prevents accidental attacks on unintended targets.

**FoxyProxy**
- Browser extension for quick proxy switching.
- Essential for toggling between normal browsing and Burp interception.

## Commands & Workflow

### Step 1 — Environment Setup
```bash
# Check Kali IP and identify target
ip addr show
nmap -sn 192.36.17.0/24
# Target: 192.36.17.3 (OWASP Mutillidae)

# Verify HTTP service
nmap -sV 192.36.17.3
# Result: Port 80 open
```

### Step 2 — Launch Burp Suite
```bash
# Start Burp Suite from terminal
burpsuite

# Setup:
# 1. Temporary project → Next → Start Burp
# 2. Accept out-of-date warning (common in lab environments)
```

### Step 3 — Configure Browser Proxy (FoxyProxy)
```bash
# In Firefox:
# 1. Install/enable FoxyProxy extension
# 2. Add Burp proxy: 127.0.0.1:8080
# 3. Enable Burp proxy profile

# Behind the scenes:
# All browser traffic → 127.0.0.1:8080 → Burp Suite → Target
```

### Step 4 — Intercept & Forward Request
```bash
# In Burp: Proxy → Intercept ON

# In Firefox:
# 1. Navigate to http://192.36.17.3
# 2. Burp intercepts the GET request

# In Burp Intercept tab:
# GET / HTTP/1.1
# Host: 192.36.17.3
# User-Agent: Mozilla/5.0...
# Accept: text/html...

# Click "Forward" to send request to server
# Page loads in browser
```

### Step 5 — Passive Crawling (Intercept OFF)
```bash
# In Burp: Proxy → Intercept OFF
# (Allows normal browsing while Burp passively captures)

# In Burp: Dashboard
# Verify: "Live passive crawl from proxy" is ON

# In Firefox: Browse the application:
# - Login page
# - SQL Injection pages
# - User info pages
# - All submenus and links

# Burp automatically captures every request in HTTP History
```

### Step 6 — Analyze HTTP History
```bash
# In Burp: Proxy → HTTP History
# Shows all captured requests with:
# - Request method (GET/POST)
# - URL path
# - Parameters
# - Response status
# - Content length

# Example captured requests:
# GET /index.php?page=login.php
# GET /index.php?page=userinfo.php
# GET /index.php?page=sqlinjection.php
```

### Step 7 — Site Map (Target Tab)
```bash
# In Burp: Target → Site map
# Visual tree structure of discovered application:
# - 192.36.17.3
#   ├── index.php
#   ├── login.php
#   ├── userinfo.php
#   ├── sqlinjection.php
#   └── ... (all discovered pages)

# Each node shows:
# - Request/response details
# - Parameters
# - Notes
```

### Step 8 — Scope Management
```bash
# In Burp: Target → Site map
# 1. Right-click target domain → "Add to scope"
# 2. Confirm: "Avoid accumulating out-of-scope items"

# In Burp: Target → Scope
# Shows:
# - Included targets: http://192.36.17.3
# - Excluded targets: (none, or manually added)

# Result:
# - Burp only processes in-scope traffic
# - Out-of-scope requests are ignored
# - Prevents accidental attacks on wrong targets
```

## Burp Suite Proxy Workflow Diagram

```
Browser → FoxyProxy → Burp Proxy (127.0.0.1:8080) → Target Server
              ↑
         [Intercept ON]  →  Manual inspection/modification
         [Intercept OFF] →  Passive capture to HTTP History
              ↓
         Dashboard: Live passive crawl ON
              ↓
         Target → Site map (auto-populated tree)
              ↓
         Scope management (in-scope only)
```

## Key Takeaways

| Feature | Purpose | Location |
|---|---|---|
| **Intercept** | Pause/modify requests manually | Proxy → Intercept |
| **HTTP History** | Review all captured traffic | Proxy → HTTP History |
| **Site Map** | Visual tree of application structure | Target → Site map |
| **Scope** | Define engagement boundaries | Target → Scope |
| **Passive Crawl** | Auto-map while browsing | Dashboard (always on) |

- **Intercept ON** = Manual control (good for analyzing specific requests).
- **Intercept OFF** = Passive capture (good for mapping the whole app).
- **Scope is critical** — Legal and contractual requirement; never attack out-of-scope targets.
- **Passive crawling** builds the site map without sending extra requests — stealthy reconnaissance.
- **HTTP History** is your audit trail — everything you did is recorded for reporting.
- Burp Suite CE has limitations (no active scanner, throttled intruder) but proxy and passive features are fully functional.
